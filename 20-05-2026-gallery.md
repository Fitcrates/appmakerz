# Modern Gallery Carousel & Image Optimization

Upgrade the `PortableTextGallery` carousel from a basic 2010s horizontal scroller to a modern, premium carousel experience, and fix image optimization issues across all three files.

## User Review Required

> [!IMPORTANT]
> The carousel redesign is a visual-behavioral overhaul. The **grid mode** remains unchanged — only the `carousel` display mode and the lightbox modal are being modernized.

> [!WARNING]  
> The current carousel uses `overflow-x-auto` with native scrolling (no prev/next buttons, no indicators, no autoplay). This looks and feels like a 2010s mobile scroll strip. The proposed design replaces this entirely with a framer-motion-powered carousel featuring navigation, pagination dots, peek-ahead previews, and smooth spring animations.

## Analysis: Current Issues

### Carousel UX Problems
1. **No navigation controls** — users must know to scroll horizontally (no prev/next buttons in carousel mode)
2. **No slide indicators** — no dots, counter, or progress bar showing position
3. **No active slide concept** — all images are visible at equal size in a flat scroll strip
4. **No autoplay** — static until manually scrolled
5. **No peek/preview** — adjacent slides aren't hinted at visually
6. **Fixed height approach** — `h-[150px] sm:h-[250px] md:h-[350px] lg:h-[550px]` with aspect-ratio fights against responsive design

## Proposed Changes

### Gallery Carousel Redesign

#### [MODIFY] [PortableTextGallery.tsx](file:///F:/StronyInternetowe/AppCrates/DevFiles/src/components/next/PortableTextGallery.tsx)

Complete rewrite of the carousel display mode (`display === 'carousel'` branch):

**New carousel design:**
- **Active slide pattern** — one primary image shown at a time with smooth crossfade/slide transitions via `framer-motion` `AnimatePresence`
- **Peek-ahead design** — subtle glimpse of next/prev slides visible at edges on desktop (glassmorphic fade)
- **Navigation arrows** — sleek prev/next buttons that appear on hover (desktop) or always show (mobile), consistent with the existing lightbox arrow style
- **Pagination dots** — minimal dot indicators below the carousel showing current position, with scale animation on active dot
- **Slide counter** — subtle `1 / 5` counter in top-right corner
- **Autoplay** — optional auto-advance (pauses on hover/interaction), 5s interval
- **Swipe gestures** — `framer-motion` drag-to-swipe with spring physics for mobile
- **Keyboard navigation** — left/right arrow keys when carousel is focused
- **Smooth transitions** — spring-based slide animations (not CSS scroll-snap)
- **Responsive heights** — aspect-ratio-driven rather than fixed pixel heights

**Lightbox modal improvements:**
- Add thumbnail strip at the bottom of the modal for quick navigation between images
- Smoother image transition animation (slide + fade instead of instant swap)

### Image Optimization Fixes

#### [MODIFY] [PortableTextGallery.tsx](file:///F:/StronyInternetowe/AppCrates/DevFiles/src/components/next/PortableTextGallery.tsx)

| Context | Fix |
|---|---|
| Carousel images | Add `.width(1200).auto('format').quality(80)` to `urlFor()` calls |
| Carousel `sizes` | Change to `"(max-width: 640px) 100vw, (max-width: 1024px) 80vw, 1200px"` |
| Grid images (fill mode) | Add `sizes` based on column count: 1-col → `100vw`, 2-col → `50vw`, etc. |
| Grid images (auto mode) | Add `.width(1200).auto('format').quality(80)` to `urlFor()` |
| Modal/lightbox image | Change `quality={100}` → `quality={90}`, add `.width(1920).auto('format')` to `urlFor()` |

#### [MODIFY] [PortableTextComponentsServer.tsx](file:///F:/StronyInternetowe/AppCrates/DevFiles/src/components/next/PortableTextComponentsServer.tsx)

- Add `.width(1400)` to the inline image `urlFor()` call so Sanity CDN serves a constrained size instead of the full original
- The raw `<img>` tag is acceptable here (server component, no client JS), but add `width` and `height` attributes for CLS prevention
